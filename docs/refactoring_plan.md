# Laions 리팩토링 계획 개요 - 2026/05/15

## 1. 핵심 기능 (Features)
  * **AI 승리예측 대결:** LightGBM 기반의 AI 모델의 시즌 경기결과 예측과 대결, AI가 틀린 경기를 유저가 맞추면 가산점 부여
  * **AI 퀴즈:** AI로 생성된 퀴즈를 풀며 재미와 학습을 동시에 얻는 참여형 콘텐츠, npc ai도 퀴즈를 풀어 유저와 경쟁
  * **주간 팬 랭킹:** 예측과 퀴즈 점수를 합산하여 팬들 간의 순위를 보여주는 게임화(Gamification) 요소, 로그인 기반 개인화, npc ai를 경쟁 상대로 투입, ai와 랭킹 대결
  * **모드 분할(일반모드/관리자모드)**
    - **일반모드** : 오늘의 날짜를 기준으로 정규시즌/포스트시즌/오프시즌 모드로 구분, 각 모드의 컨텐츠를 제공
      - **정규시즌** : 정규시즌 전 경기의 승리 예측을 AI과 겨루는 모드
      - **포스트시즌** : 정규시즌과 달리 포스트시즌은 한경기만 치룸, 마찬가지로 포스트시즌의 승리 예측 + AI가 예측한 다음시리즈 진출팀 표시
      - **오프시즌** : 이번 시즌의 전체 결과를 반영하여 시뮬레이션으로 다음 시즌의 순위 예측을 출력
    - **관리자모드** : 일반모드는 오늘 날짜를 기준으로 시스템이 구동, 다른 날짜의 시스템이 제대로 구현되는지를 확인하기 위해 강제로 날짜를 지정하는 모드, 날짜를 오늘 날짜가 아닌 관리자모드에서 지정한 날짜 기준으로 시스템 파이프라인을 실시하여 전체 점검이 목표
  * **오늘의 삼성라이온즈 역사:** 당일을 기준으로 과거의 삼성 라이온즈 역사 출력

## 2. 기존 방식 - 2026/05/08 이전
삼성 라이온즈만의 경기 사용, 포스트시즌의 경우 삼성이 진출하면 삼성의 다음시리즈 진출 확률계산(포스트시즌 일정 스크래핑이 어려워 이 방식으로 대체했었음), 삼성이 진출 실패 시 바로 오프시즌으로 넘어감, 관리자 모드에 기능들이 존재하여 일반 모드에서 관리자 모드를 참조하는 기이한 현상 발생, 기존에 사용하던 db는 내 개인 pc에 설치된 postgresql을 사용했음, postgresql이 없는 환경에선 사용불가, 로그인 기능을 firebase로 구현, 기존 퀴즈 생성 AI는 일정량 무료라 성능이 나빠도 gemini flash를 사용했으나 현재 무료 기능이 상실된듯, 모델학습의 경우 2021~2024를 학습하고 2025를 검증셋으로 사용, 관리자 모드 전용 db도 있었음 

## 3. 주요 변경점 및 변경 계획
- 전 경기 예측으로 확장, 포스트 시즌의 경우 삼성 라이온즈만이 아닌 당일 시리즈 일정의 경기 예측으로 변화
- 관리자 모드를 오로지 날짜만 강제 주입하는 걸로 변경한 후 기존 모드를 작동하게 하는 방식으로 리팩토링, 트랜젝션 롤백 기능을 사용하여 db를 분리하지 않으면서 오염이 되지 않게 사용
- 모델 학습의 경우 검증 기능이 불필요하고 데이터셋을 더 많이 확보하는게 낫다 판단하여 학습 방식을 기존 2025를 검증셋으로 사용하는것에서 2025까지 모두 학습하는걸로 변경
- 현재 리그 순위도 출력하여 삼성 팬 사이트인만큼 삼성의 순위를 강조하는 식으로 구현, 순위는 스크래핑은 힘드니 기존에 저장된 db로 순위를 계산하는 방식 사용
- 퀴즈 생성기 또한 프롬포트 정교화와 스스로 논리 점검하는 로직 추가로 보강, gemini가 유료라 하더라도 검색기능을 포함하고 있어서 모델 변경도 계획했으나 유지하기로 결정
- 스케쥴러를 만들지 않았으나 이번 기회에 만들 예정, github actions를 사용해 기기가 꺼져도 작동이 되도록 구현
- 기존 db 사용과 firebase 기능을 supabase로 단일화
- 도커화 하여 배포 용이하도록 구현?
- 전체 경기를 다루는것으로 변경하다보니 삼성의 색깔이 약해짐, 삼성 라이온즈 역사 섹션 추가
- ai의 최근 경기 예측 정확도를 표현하는 ai 퍼포먼스 섹션의 구조적 결함 발견, 이를 삭제하고 npc ai를 만들어 유저 랭킹에 투입하여 경쟁하기
- 기존에 사용하던 스크래핑 방식인 kbo 홈페이지에 /ws/ api 사용이 robots.txt에서 금지하는 걸 발견함 -> 난이도가 낮고 제한이 적은 다음 스포츠 스크래핑으로 변경

## 4. 기능별 상세 설명
- **스케쥴러:** 매일 새벽 1시 기준으로 전날까지의 경기 결과 스크래핑 + 오늘 경기 일정 스크래핑, 퀴즈 생성
- **AI 승리예측 대결:** LightGBM 기반의 2021~2025년까지의 정규시즌 경기를 elo 레이팅으로 학습한 모델이 오늘 날짜의 전경기 승패 예측을 함, 팬들도 마찬가지로 승패 예측을 하여 AI가 틀린걸 팬이 맞추면 더 높은 점수를 부여
- **AI 퀴즈:** 삼성 라이온즈에 관련된 퀴즈를 난이도별로 분류하여 미리 AI에게 작성하도록 하여 db에 저장, ai가 만들자마자 웹에 표시하는건 시간이 오래 걸려 미리 만들고 불러오는 방식, 갯수 제한이 없으면 주간 팬 랭킹에 영향이 가기에 난이도별 퀴즈를 매일 하나씩만 출력, 맨 처음 퀴즈를 만들때 1982~2025년의 퀴즈를 난이도별로 대량 생산하여 db로 저장, 2026년 경기가 진행중이니 매일 or 매주 2026년을 주제로한 퀴즈를 스케줄러에서 생성
- **주간 팬 랭킹:** 승리예측 점수와 퀴즈 점수의 합산으로 주간단위 높은 점수의 유저 랭킹 출력, 매주 월요일 리셋
- **AI NPC:** AI를 하나의 유저인걸로 취급, AI가 승리예측에서 맞춘 점수 + 별도의 퀴즈 풀이 AI를 만들어 퀴즈 점수를 합산하여 실제 유저와 경쟁하여 주간 팬 랭킹 섹션에 랭킹을 매김
- **리그 순위:** 매일 스크래핑 된 DB를 정리하여 순위 테이블 생성하여 출력, 삼성 라이온즈의 순위에 강조점 부여
- **관리자 모드:** 기본 모드는 오늘 날짜의 경기 기준으로 컨텐츠를 결정, 그러나 테스트 용도 및 포트폴리오 용도 등은 오늘 날짜만 한정해서 보여주게 되면 다른 모드의 기능을 검증 및 시연하기엔 부족, 오늘 날짜가 아닌 강제적으로 다른 날짜를 부여하는 관리자 모드, 트랜젝션 롤백 기능을 사용하여 db를 분리하지 않으면서 오염이 되지 않게 사용
- **일반 모드** 
  - 정규시즌 모드 : 어제 경기까지의 결과를 업데이트 및 오늘 경기 일정을 스크래핑, 오늘 펼쳐질 경기의 승리를 모델이 예측, 유저도 승리 예측을 하여 모델이 틀린 경기를 유저가 맞추게 되면 유저에게 가산점을 부여
  - 포스트시즌 모드 : 정규시즌 전경기(720경기)가 종료되면 포스트시즌 모드로 돌입, 포스트시즌 일정을 스크래핑하여 정규시즌 모드와 마찬가지로 모델과 유저의 예측 대결, 시뮬레이션을 거쳐 AI가 다음 시리즈 진출팀 예상
  - 오프시즌 모드 : 한국시리즈 4경기 승리팀이 나올경우 오프시즌 모드로 돌입, 지금껏 학습했던 기간 + 이번시즌의 결과를 새로 모델에 학습시킨 후 다음 시즌의 순위를 시뮬레이션을 통해 예측
- **오늘의 삼성라이온즈 역사:** 당일의 과거 연도에 일어난 삼성 라이온즈의 역사적 사실을 미리 AI로 수집한 다음 DB에 저장하여 날짜마다 출력 (예: 오늘이 2026년 10월 2일일때, 역사 섹션에 '2003년 오늘 이승엽 선수가 56호 홈런을 달성했습니다.' 같은 삼성 라이온즈의 역사적 사실을 출력)  

## 5. UI 간략 구조
- 기본 모드의 세부모드에 따라 현재 날짜 기준으로 UI를 출력(예: 오늘 날짜로 정규시즌이 진행 중이면 정규시즌 모드 UI 출력)
- 관리자 모드는 점검 및 시연이 목적이기에 기본 모드와 UI는 동일, 관리자 모드에서 설정한 날짜를 기준으로 출력

- 모든 모드에 공통 출력 섹션 :
  - 퀴즈 섹션 : 난이도 상중하를 선택해서 오늘의 퀴즈 출력
  - 주간 팬 랭킹 섹션 : 야구가 없는 월요일에 초기화, 주간 팬 랭킹 점수와 순위를 출력, npc도 팬과 마찬가지로 같이 랭킹 표시
  - 오늘의 삼성라이온즈 역사 섹션 : 당일 날짜의 역사 출력
  
- 정규시즌 모드
  - 승리 예측 섹션 : 오늘 날짜 경기 일정 + 모델의 승리 예측 팀 표시, 유저가 승리예측팀을 선택하여 아래에 제출 버튼으로 오늘의 예측 제출
  - 현재 정규시즌 리그 순위 : 리그 순위를 표시, 삼성 라이온즈를 강조, 새로 스크래핑 하는건 불확실하여 기존에 수집했던 데이터를 기반으로 리그 순위 테이블 만든 후 출력

- 포스트시즌 모드
  - 오늘의 경기 승리 예측 섹션 : 현재 진행중인 시리즈(예: 와일드카드 결정전)를 표시, 해당 경기의 승부 예측, 다음 시리즈 진출 예상 팀 출력(예 : 준플레이오프 진출 예상 팀 '한화 이글스')
  - 포스트시즌 대진표 섹션 : 대진표 출력

- 오프시즌 모드
  - 다음시즌 순위 예측 섹션 : 이번 시즌까지의 결과를 반영한 다음 시즌 순위 예측
  - 지금 시즌 순위 섹션 : 이번 시즌 순위와 다음 시즌 예측 순위를 비교하기 위함

# 6. 정기적 수집과 비정기적 수집의 분리
- 퀴즈, 역사, 경기 정보 등을 정기적으로 수집하는 정보들과 그렇지 않은 정보들을 분리하여 매일 수집이 필요한 데이터들은 스케쥴러로, 그렇지 않은 데이터는 stack_service 폴더에 모음
- 정기적 수집 정보 : 전날과 당일 경기 정보 매일 수집, 최신 퀴즈 데이터(2026년 기록 등) 매주 수집
- 비정기적 수집 정보 : 과거의 퀴즈 데이터(1982~2025년 퀴즈), 과거의 경기 정보(2007~2025), 1년 해당일의 역사 정보, 초기모델 학습


# 7. 보완점
- 트랜잭션 롤백이 난이도가 높음, 관리자용 DB를 만드는 방식을 부활할지 고민
- 우천취소, 더블헤더, 무승부 등의 특수 상황이 반영 되어있는지 확인
