기본적으로 이 시스템은 당일 날짜를 기준으로 스크래핑과 경기 일정 등을 결정하는 시스템이야
야구는 한 팀당 144경기를 하고 10개 팀이 총 720경기를 한 시즌에 치뤄, 오늘이 4월 24일이니 (스케쥴러는 미구현) 매일 0시에 전날까지의 경기 결과를 스크래핑하여 db에 저장을 하고 오늘의 일정을 가져와서 이 일정을 기준으로 예측을 하는 시스템이야, 정규시즌 720경기가 모두 종료되면 포스트시즌으로 넘어가는게 야구고 이 포스트시즌은 한국시리즈 4승을 하여 우승을 한 팀이 생기면 끝나, 포스트시즌이 끝나면 오프시즌이 되어 야구는 아무것도 안해, 이 시스템에서 오프시즌에 할 컨텐츠로 다음시즌 순위 예측을 준비한거야
오프시즌의 경우 내년 시즌의 경기를 예측하기 위해서 과거 데이터만을 학습했었던 모델이 계속 과거 데이터만으로 예측을 하면 정확성이 낮아지니 이번 시즌이 끝났으니 이번시즌 전체 경기 결과를 추가적으로 새로 학습을 하는거야

이게 기본적인 시스템이고 다시 말하지만 이 시스템의 기준은 오늘 날짜야

관리자 모드를 왜 만들었냐면 예를 들어서 오늘 이 시스템을 구동시킨다면 현실의 물리적인 시간으로 정규시즌이 시작된지 1달도 채 되지 않아 각팀당 20경기 정도만을 수행한 상황이야 이런 상황에선 죽었다 깨어나도 720경기 이후 포스트시즌의 파이프라인을 테스트 할 수 없고 오프시즌의 파이프라인을 볼 수 없어 그래서 물리적 시간을 무시하고 오늘을 admin date로 가정한 채 시스템을 굴리는 목적으로 만든게 admin 모드야


관리자 모드의 목적은 오늘 날짜 기준으로 굴러가는 시스템의 물리적 시간을 다른 날짜로 설정하여 기본적인 시스템이 제대로 굴러가는지를 점검하기 위함이야




**api리스트(무조건 따를 필요는 없지만 단 한가지 절대 무슨일이 있어도 admin모드에 기능을 넣는게 아니다라는건 무조건 필수로 정확하게 지켜야한다)**
- /api/crawler/historical(특정 연도를 범위로 설정하여 수집)
- /api/crawler/daily-update(당일 날짜를 기준으로 수집)

- /api/features/rebuild(수집한 데이터들을 모델이 학습에 사용할 피처로 재구성)

- /api/model/prediction(당일 날짜의 전체 경기 승리 예측)
- /api/model/retrain(offseason 모드의 경우 당해시즌 경기까지의 데이터가 이미 수집 되었으니 rebuild를 통해 피처로 만든 후 재학습)

- /api/simulation/projection(retrain으로 학습된 모델이 시뮬레이션을 거쳐 다음 시즌의 순위 예측)

- /api/performance(모델의 최근 10경기 적중률 계산)

- /api/user/ranking(승리예측과 퀴즈를 통해 얻은 점수로 유저들의 랭킹 출력)
- /api/user/reset(공정성을 위해 월요일 0시에 유저 점수 초기화) 

- /api/admin/forcedate(특정 날짜 강제 지정 후 그 날짜의 전체 파이프라인이 작동하도록 사용, 여기에 기능을 넣게 되면 일반적 상황에서 여기를 참조해야하니 여기에 다른기능을 넣을게 아니고 여기서는 현재날짜를 무시하고 특정 날짜를 현재날짜로 인식하게만 하면 되는것, 여기서도 admin crawler,feature,model,simulatin,performance,ranking,reset등의 기능을 넣는게 아니고 이걸 몇시간째 설명중인데 왜 정확하다고 지랄인지 모르겠네)
